Данные задачи
1. Входные и выходные данные организованы в виде файлов .json.
2. Последовательность доступных купюр приводится в кошельке
"wallet", который представляет собой массив пар [номинал,
доступное количество]. Номинал и количество есть целые,
положительные числа в диапазоне [1; 5 000]. Номиналы не
повторяются.
3. Запрашиваемая сумма "amount" является целым положительным
числом. Запрашиваемая сумма требует не более 15000 купюр
различного номинала.
4. Стратегия "strategy" является критерием выбора набора купюр,
является строкой, имеет три возможных значения
"MAX" – требуется наибольшее количество купюр старшего
номинала,
"MIN" – требуется наибольшее количество купюр младшего
номинала.
5. Результат расчета должен представлен в объекте
"dispense", аналогичном по структуре "wallet". В "dispense"
приводятся пары [номинал, количество] для купюр в выбранном
наборе, номиналы не повторяются, номиналы с нулевым
количеством не приводятся.
